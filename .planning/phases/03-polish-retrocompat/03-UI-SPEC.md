### Exceptions

| Value | Location | Justification |
|-------|----------|---------------|
| `x=2` | Badge left margin from poster edge (`place(x=2, ...)`) | Codebase-inherited coordinate. Matches existing ✓ VISTO / ⏳ PDTE badge pattern at `Videoclub` line 1393 (`place(x=2, y=2)`). The `.place()` coordinate system in Tkinter works in raw pixels; aligning to the existing badge x-origin takes precedence over the 4-point grid. This value must not be changed to 4 — doing so would misalign the new badge relative to the established badge column. |

Specific badge coordinates:
- `x=2` — left edge, 2px from poster zone left edge (matches ✓ VISTO / ⏳ PDTE badge x position)
- `y=POSTER_H - 18` — bottom-left of poster zone (`POSTER_H = 200`, so `y=182`)
- This leaves the top-left area (`y=2`) clear for status badges (VISTO/PDTE) and top-right for type badge

## Component Inventory

### GRID-01: Episode Progress Badge (new widget in ContentCard._build)

Location: `poster_zone` (the `tk.Frame` at lines 1378–1410 of `Videoclub`)

Condition: render only when `item["type"] == "series"` AND `watched_count >= 1`.

Widget:
```python
tk.Label(
    poster_zone,
    text=f"{watched_count} ep",
    bg=C["seen"], fg="white",
    font=(FONT_FAMILY, 7, "bold"),
    padx=3
).place(x=2, y=POSTER_H - 18)
```

- `padx=3` matches existing badge padx (lines 1394–1400)
- `y=POSTER_H - 18` = `y=182` — bottom-left, clear of top-left status badges at `y=2`
- Badge text format: `"N ep"` (e.g. `"4 ep"`, `"12 ep"`) — NOT "N eps", NOT "N episodios"
- Show badge AFTER rendering the poster image and existing top-corner badges — order matters for z-index (`.place()` later = on top)

Data source: new DB method `get_total_watched_count(user_id, item_id) -> int` (or inline `SELECT COUNT(*) FROM episode_progress WHERE user_id=? AND item_id=? AND watched=1`). The existing `get_season_progress()` is per-season and cannot be used without knowing the seasons list — not suitable here.

Refresh: `ContentCard._build()` is called on every grid rebuild. `SeasonViewWindow._on_close` already calls `on_refresh()` → grid rebuilds → badge auto-updates. No extra wiring needed.

### BACK-01/02: Disabled "Ver temporadas" Button (DetailWindow._build)

The existing block at `Videoclub` line 1575–1585:
```python
tmdb_id = self.item.get("tmdb_id") or ""
if itype == "series" and tmdb_id:
    tk.Button(...enabled button...).pack(fill="x", pady=(8, 0))
```

Must become:
```python
tmdb_id = self.item.get("tmdb_id") or ""
if itype == "series":
    if tmdb_id:
        # enabled button — existing code unchanged
        tk.Button(
            left, text="📺 Ver temporadas",
            bg=C["accent"], fg="white",
            font=(FONT_FAMILY, 9), relief="flat", cursor="hand2",
            command=lambda: self._open_seasons(
                self.item["tmdb_id"],
                self.item.get("num_seasons", 0)
            )
        ).pack(fill="x", pady=(8, 0))
    else:
        # disabled button — new code
        btn = tk.Button(
            left, text="📺 Ver temporadas",
            bg=C["surface"], fg=C["muted"],
            font=(FONT_FAMILY, 9), relief="flat",
            state="disabled"
        )
        btn.pack(fill="x", pady=(8, 0))
        Tooltip(btn, "Re-añade la serie via búsqueda para activar")
```

Visual differences between enabled and disabled state:

| Property | Enabled (tmdb_id present) | Disabled (tmdb_id NULL) |
|----------|--------------------------|------------------------|
| `bg` | `C["accent"]` `#e50914` | `C["surface"]` `#111118` |
| `fg` | `"white"` | `C["muted"]` `#888899` |
| `cursor` | `"hand2"` | default (no pointer — `state=disabled` removes it) |
| `state` | normal | `"disabled"` |
| `command` | `_open_seasons(...)` | none — state=disabled blocks all click events |
| Tooltip | none | `"Re-añade la serie via búsqueda para activar"` |

Click behavior on disabled button: `state="disabled"` absorbs click — nothing happens.
Tooltip is the complete feedback mechanism (BACK-02 satisfied by tooltip).

Source: 03-CONTEXT.md D-08, D-09, D-10, D-11.

## Interaction Contract

### GRID-01: Badge Visibility Protocol

1. `ContentCard._build()` is called (on grid build or rebuild).
2. Query: `db.get_total_watched_count(user_id, item_id)` → returns `int`.
3. If `item["type"] == "series"` AND `watched_count >= 1`: render badge label at `place(x=2, y=182)`.
4. If `watched_count == 0` OR `item["type"] != "series"`: do NOT render badge (no placeholder, no empty label).
5. Badge is a static label — no hover, no click, no tooltip.
6. After any `SeasonViewWindow` close: `on_refresh()` → full grid rebuild → badge re-queries DB → reflects new count automatically.

### BACK-01/02: Disabled Button Protocol

1. `DetailWindow._build()` calls `self.item.get("tmdb_id") or ""`.
2. If `itype == "series"` AND `tmdb_id` is truthy: render enabled button (existing behavior, unchanged).
3. If `itype == "series"` AND `tmdb_id` is falsy (NULL or empty string): render disabled button + `Tooltip`.
4. User hovers disabled button: tooltip appears with `"Re-añade la serie via búsqueda para activar"`.
5. User clicks disabled button: `state="disabled"` absorbs click — nothing happens.
6. No dialog, no toast, no redirect. Tooltip is the complete interaction.

## Critical Implementation Guards

1. **Badge only for `type == "series"`** — movie cards never get a progress badge regardless of any hypothetical future watched-count logic.
2. **Badge bottom-left (`y=POSTER_H-18`)** — never use `y=2` (reserved for status badges VISTO/PDTE at top-left). Z-order: `.place()` the badge after all other poster widgets to ensure it renders on top.
3. **`get_total_watched_count` queries across all seasons** — do not use `get_season_progress` (per-season only). The badge shows total across the series.
4. **Disabled button has no `cursor="hand2"`** — `state="disabled"` already removes pointer cursor; do not fight Tkinter by trying to set it.
5. **Disabled button has no `command`** — `state="disabled"` blocks clicks; do not add a `command` that would fire on programmatic `.invoke()`.
6. **Tooltip attaches to the `btn` widget reference** — call `Tooltip(btn, "...")` after `btn` is assigned; cannot attach to an anonymous widget.
7. **`tmdb_id` check uses `or ""`** — `self.item.get("tmdb_id") or ""` coerces both `None` (SQL NULL) and `""` (empty string) to falsy. The guard must match line 1575's existing pattern.

Source: 03-CONTEXT.md D-02, D-03, D-05, D-07, D-08, D-09, D-10.

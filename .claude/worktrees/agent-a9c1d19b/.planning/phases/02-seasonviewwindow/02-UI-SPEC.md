# Phase 2 — UI Design Contract: SeasonViewWindow

> Visual and interaction contract for SeasonViewWindow — the modal Toplevel that shows
> season tabs and episode checkboxes for a TV series.
> Generated by gsd-ui-researcher. Consumed by gsd-planner and gsd-executor.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom dark theme via color dict `C` |
| Preset | not applicable |
| Component library | Tkinter (tk + ttk) — no third-party widget library |
| Icon library | Unicode emoji inline in button/label text (existing app pattern) |

## Spacing Scale

Tkinter uses pixel values. The app has no formal token system but follows a consistent
rhythm derived from code inspection. The following values are declared for Phase 2:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap inside episode rows; checkbox left padding |
| sm | 8px | Episode row inner padding (padx); section separator height |
| md | 16px | Window outer padding (padx/pady on main frame) |
| lg | 24px | Vertical breathing room above/below season header block |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Episode row height is content-driven (no fixed height); `pady=2` between rows keeps density high.
- Season header uses `pady=(6, 4)` for asymmetric spacing (tighter below than above).
- ttk.Notebook tab padding: `[10, 4]` (horizontal, vertical) — from ARCHITECTURE.md Style override.
- Touch target minimum: checkboxes are 20×20px minimum; Tkinter Checkbutton default satisfies this on 96 DPI.

Source: ARCHITECTURE.md episode row code examples; existing `DetailWindow._build()` padding patterns at lines 1525–1530.

## Typography

All text uses `FONT_FAMILY` ("Segoe UI"). Tkinter font tuples: `(FONT_FAMILY, size)` or `(FONT_FAMILY, size, "bold")`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|

## Color

All values from the existing `C` dict at `Videoclub` lines 117–143.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0f` (`C["bg"]`) | Window background, canvas background, inner Frame background |
| Secondary (30%) | `#111118` (`C["surface"]`) | Episode row background; season header background; notes text area |
| Card / tertiary | `#18181f` (`C["card"]`) | Not used in this phase — SeasonViewWindow uses surface, not card |
| Border / separator | `#2a2a35` (`C["border"]`) | Horizontal rule between season header and episode list; air date text |
| Accent (10%) | `#e50914` (`C["accent"]`) | "Ver temporadas" button in DetailWindow (entry point only) |
| Muted | `#888899` (`C["muted"]`) | Progress counter text; loading/error labels; future episode text (disabled) |
| Text | `#f0f0f5` (`C["text"]`) | Episode title for watched episodes; enabled episode text |
| Seen / Complete | `#1db954` (`C["seen"]`) | Season header accent when 100% of episodes are watched; checkmark color when full season complete |
| Pending | `#f4a261` (`C["pending"]`) | Not used in SeasonViewWindow — watch status is episode-level, not series-level here |
| Hover | `#22222d` (`C["hover"]`) | Episode row hover state background |
| Tooltip | `#1e1e28` (`C["tooltip"]`) | Tooltip background for the "BACK-01" disabled button (Phase 3 concern) |
| Destructive | `#3d0000` / `#ff6b6b` | Not used in Phase 2 (no delete actions in SeasonViewWindow) |

**Accent reserved for:** the "Ver temporadas" entry-point button in DetailWindow only. Inside SeasonViewWindow itself, accent is NOT used — the window interior uses surface/muted/seen for its color language.

**Future episodes (air_date > today):** Use `C["muted"]` (`#888899`) for text color. Checkbox `state=DISABLED`. This is a read-only state indicator, not a semantic color — no new color token needed.

**Season tab theming (ttk.Notebook):** The default ttk light-grey must be overridden. Use this Style configuration (from ARCHITECTURE.md, PITFALLS.md):

```python
style = ttk.Style()
style.theme_use("default")
style.configure("Dark.TNotebook",     background=C["bg"])
style.configure("Dark.TNotebook.Tab", background=C["surface"],
                foreground=C["muted"], padding=[10, 4])
style.map("Dark.TNotebook.Tab",
          background=[("selected", C["bg"])],
          foreground=[("selected", C["text"])])
```

Apply style name `"Dark.TNotebook"` to the Notebook widget. This is the only ttk Style override needed in this phase. All other widgets use plain `tk` with `bg=C[...]` kwargs.

## Component Inventory

### SeasonViewWindow (tk.Toplevel)

- `configure(bg=C["bg"])`
- `resizable(True, True)` — user may want to see more episodes
- `grab_set()` on `__init__` — takes modal focus from DetailWindow
- `bind("<Escape>", lambda e: self.destroy())`
- `protocol("WM_DELETE_WINDOW", self._on_close)` — calls `on_refresh` before destroy
- `_center()` helper — centers on screen (same pattern as DetailWindow._center())
- Title: `f"Temporadas — {series_title[:50]}"`
- Window minimum size: 600×480 (seasons list + episode list must be usable)

### Season Tab Strip (ttk.Notebook)

- Style: `"Dark.TNotebook"` (see Color section)
- Tab labels: TMDb `name` field from the `seasons[]` array; fallback `"Especiales"` for season_number=0 with empty name; fallback `f"T{season_number}"` for numbered seasons
- One `tk.Frame(bg=C["bg"])` per tab — populated lazily on `<<NotebookTabChanged>>`
- `_loaded_seasons: set[int]` tracks which tabs have been fetched
- `_season_frames: dict[int, tk.Frame]` maps season_number to tab frame
- Trigger first tab load immediately after binding: `self.after(0, self._on_tab_changed)`

### Season Header (per tab, inside tab frame)

Layout: horizontal strip at top of tab frame, `bg=C["surface"]`, `padx=8, pady=6`.

Left side:
- Season name label — `font=(FONT_FAMILY, 10, "bold")`, `fg=C["text"]`
- Progress counter label — `font=(FONT_FAMILY, 9)`, `fg=C["muted"]`
  - Format: `"X / Y episodios vistos"`
  - Updates immediately on every checkbox toggle (in-place `.config(text=...)`)
  - When X == Y (all watched): `fg=C["seen"]` — season complete visual feedback

Right side (right-aligned):
- "Marcar toda la temporada" button
  - `bg=C["surface"]`, `fg=C["muted"]`, `font=(FONT_FAMILY, 9)`, `relief="flat"`, `cursor="hand2"`
  - On click: bulk-insert all episodes as watched → refresh all BooleanVars → update progress counter
  - Button text on complete season: does NOT change to "Desmarcar" in v1 (out of scope)

Horizontal separator: `tk.Frame(bg=C["border"], height=1)` between header and episode list.

### Scrollable Episode List (Canvas + Frame)

- `tk.Canvas(bg=C["bg"], highlightthickness=0)` — no border ring
- `tk.Scrollbar(orient="vertical")` — right side
- `inner = tk.Frame(canvas, bg=C["bg"])` — populated with episode rows
- `canvas.create_window((0, 0), window=inner, anchor="nw")`
- `inner.bind("<Configure>", lambda e: canvas.configure(scrollregion=canvas.bbox("all")))`
- MouseWheel binding on BOTH canvas AND inner Frame (Windows requirement):
  ```python
  for w in (canvas, inner):
      w.bind("<MouseWheel>",
             lambda e: canvas.yview_scroll(-1 * (e.delta // 120), "units"))
  ```
- Canvas fills tab frame: `canvas.pack(side="left", fill="both", expand=True)`
- Scrollbar: `scrollbar.pack(side="right", fill="y")`

Source: ARCHITECTURE.md §Question 6; PITFALLS.md §Canvas scroll on Windows.

### Episode Row (per episode, inside inner Frame)

Layout: `tk.Frame(inner, bg=C["surface"], pady=2)`, `pack(fill="x", padx=8, pady=2)`

Left to right:
1. `tk.Checkbutton` — `bg=C["surface"]`, `fg=C["text"]`, `activebackground=C["surface"]`, `selectcolor=C["bg"]`, `relief="flat"`, `cursor="hand2"`, `variable=BooleanVar`
   - `state=DISABLED` + `fg=C["muted"]` for future episodes (air_date > today)
   - `command=_toggle` — writes to SQLite immediately, updates progress counter
2. Episode number + title label — `text=f"{ep_num:02d}. {ep_name}"`, `pack(side="left", fill="x", expand=True)`
   - Watched: `fg=C["text"]`, font normal
   - Unwatched (past): `fg=C["muted"]`, font normal
   - Future (unaired): `fg=C["muted"]`, font normal — visually identical to unwatched; checkbox disabled is the key distinction
3. Air date label — `text=ep_date[:7]` (YYYY-MM format), `fg=C["border"]`, `font=(FONT_FAMILY, 8)`, `pack(side="right", padx=6)`
   - Omitted if `ep_date` is empty string

Hover: `row.bind("<Enter>", lambda e: row.config(bg=C["hover"]))` + `row.bind("<Leave>", lambda e: row.config(bg=C["surface"]))`

Source: ARCHITECTURE.md §Question 6 episode row helper; FEATURES.md §Episode row layout.

### Loading State (per tab)

Shown immediately when tab is selected and not yet loaded:
- `tk.Label(tab_frame, text="Cargando temporada...", bg=C["bg"], fg=C["muted"], font=(FONT_FAMILY, 10))`
- `.pack(pady=20)`
- Destroyed in `_on_done()` callback (always via `self.after(0, ...)` — never from worker thread)

### Error State (per tab)

Shown when `fetch_tmdb_season_episodes` returns `{}`:
- `tk.Label(tab_frame, text="No se pudo cargar la temporada.", bg=C["bg"], fg=C["muted"], font=(FONT_FAMILY, 10))`
- `.pack(pady=20)`

For HTTP 404 (season not yet in TMDb):
- `text="Esta temporada aún no tiene datos en TMDb."`

For network/other errors:
- `text="Error al conectar con TMDb. Comprueba tu conexión."`

Source: REQUIREMENTS.md UI-09; FEATURES.md §Table Stakes "Error state if TMDb fails".

### "Ver temporadas" Button (in DetailWindow)

This is Phase 2's entry point — a new button added to the existing `DetailWindow._build()` left column.

- Visible only when: `itype == "series"` AND `tmdb_id` is non-empty string
- Position: below the "🗑 Eliminar" button in the left panel
- Style: `bg=C["accent"]`, `fg="white"`, `font=(FONT_FAMILY, 9)`, `relief="flat"`, `cursor="hand2"`
- Text: `"📺 Ver temporadas"`
- Command: `lambda: self._open_seasons(tmdb_id, num_seasons)`
- `pack(fill="x", pady=(6, 0))`

For series with `tmdb_id = NULL` (Phase 3 concern — BACK-01/02): button is NOT shown in Phase 2. The guard `if tmdb_id:` is sufficient.

Source: ARCHITECTURE.md §Question 1; REQUIREMENTS.md UI-01.

## Interaction Contract

### Modal Stack Protocol

1. `DetailWindow` has `grab_set()` active.
2. User clicks "Ver temporadas" → `DetailWindow._open_seasons()` instantiates `SeasonViewWindow`.
3. `SeasonViewWindow.__init__` calls `grab_set()` — takes grab from DetailWindow automatically.
4. While `SeasonViewWindow` is open: DetailWindow is non-interactive (modal blocked).
5. `DetailWindow._open_seasons()` binds: `win.bind("<Destroy>", lambda e: self.grab_set() if self.winfo_exists() else None)`
6. User closes `SeasonViewWindow` (Escape, X button, or programmatic `destroy()`):
   - `_on_close()` calls `self.on_refresh()` (propagates to grid).
   - `self.destroy()` fires.
   - `<Destroy>` binding on win fires → `DetailWindow.grab_set()` restores modal focus.

Source: ARCHITECTURE.md §Question 1; PITFALLS.md §grab_set re-entry.

### Lazy Loading Protocol

1. `_build_notebook()` creates N tab frames (empty). Does NOT fetch any data.
2. Binds `<<NotebookTabChanged>>` → `_on_tab_changed()`.
3. `self.after(0, self._on_tab_changed)` triggers load of first tab immediately after init.
4. `_on_tab_changed()`: reads current tab index → maps to season_number → checks `_loaded_seasons`.
5. If not loaded: add to `_loaded_seasons`, call `_load_season(season_number, frame)`.
6. `_load_season()`: shows spinner (main thread) → submits worker to `_season_pool`.
7. Worker fetches via `fetch_tmdb_season_episodes(tmdb_id, season_number)` (lru_cached).
8. Worker calls `self.after(0, lambda d=data: _on_done(d))`.
9. `_on_done()`: checks `self.winfo_exists()` → destroys spinner → renders episode list or error.

Season numbers come from TMDb `seasons[]` array stored in item dict or re-fetched via `fetch_tmdb_detail(tmdb_id, "tv")` (lru_cached). Iterate `s["season_number"]` — never `range(1, num_seasons+1)`.

Source: ARCHITECTURE.md §Question 3; PITFALLS.md §C1.

### Checkbox Toggle Protocol

1. User clicks checkbox → `BooleanVar` toggles.
2. `_toggle()` fires synchronously on main thread.
3. `db.set_episode_watched(user_id, item_id, season_number, ep_num, new_state)` — SQLite upsert.
4. Progress counter label updated in-place: `prog_lbl.config(text=f"{watched_count} / {total} episodios vistos")`.
5. If `watched_count == total`: `prog_lbl.config(fg=C["seen"])` — season complete.
6. No confirmation dialog — action is trivially reversible by re-clicking.

Source: REQUIREMENTS.md UI-06; FEATURES.md §Standard Interaction.

### "Marcar toda la temporada" Protocol

1. User clicks season header button.
2. Iterate all episodes in `_season_cache[season_number]["episodes"]`.
3. For each episode: `db.set_episode_watched(user_id, item_id, season_number, ep_num, True)`.
4. Set all `BooleanVar` values to `True`.
5. Update all episode row label colors to `fg=C["text"]` (watched state).
6. Update progress counter to `"Y / Y episodios vistos"` with `fg=C["seen"]`.
7. No undo/partial state — bulk operation is atomic from user's perspective.

Source: REQUIREMENTS.md UI-07; FEATURES.md §"Mark entire season watched" button.

### Window Close / Refresh Protocol

1. `protocol("WM_DELETE_WINDOW", self._on_close)` handles X button.
2. `bind("<Escape>", lambda e: self._on_close())`.
3. `_on_close()`:
   ```python
   def _on_close(self):
       self._destroyed = True
       if callable(self.on_refresh):
           self.on_refresh()
       self.destroy()
   ```
4. `on_refresh` is the grid-level refresh callback passed from ContentCard → DetailWindow → SeasonViewWindow.
5. All `after()` callbacks guard with `if self._destroyed or not self.winfo_exists(): return`.

Source: PITFALLS.md §M3 state desync; §m4 after() on destroyed Toplevel; REQUIREMENTS.md UI-10.

## Critical Implementation Guards

These are not visual but must be enforced by the executor:

1. **No UI calls from worker thread.** All widget mutations via `self.after(0, callback)`. Checkbutton `_toggle()` runs on main thread — safe. Episode list render runs on main thread via `after()`.
2. **`winfo_exists()` in every `after()` callback.** Window may be closed while fetch is in progress.
3. **`self._destroyed` flag.** Set in `_on_close()`. Checked in `_on_done()` before rendering.
4. **Season numbers from TMDb `seasons[]` array, not `range()`.** Prevents Season 0 gap and irregular numbering bugs.
5. **`episode_number` from TMDb field, not list index.** PK component in `episode_progress`.
6. **MouseWheel on both Canvas and inner Frame.** Inner Frame events do not bubble to Canvas on Windows.
7. **ttk.Notebook Style applied before notebook creation.** `style.theme_use("default")` must precede `ttk.Notebook(self, style="Dark.TNotebook")`.

Source: PITFALLS.md C1, C4, M2, m4; ARCHITECTURE.md §Key Constraints.
